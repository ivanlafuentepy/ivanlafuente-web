1 · ¿Para qué sirve un Proyecto?

La pregunta más simple:
¿qué es un proyecto?

Una carpeta. Con un archivo adentro.
Eso es todo.

Lo que lo hace poderoso no es la carpeta — es lo que ponés en ese archivo: el CLAUDE.md.

mi-proyecto
mi-proyecto/
└── CLAUDE.md

# Lo primero que la IA lee al abrir.
# Antes de que escribas una palabra,
# ya sabe el contexto completo.
La analogía que lo explica todo

El CLAUDE.md es el manual del empleado nuevo

Se lo entregás a alguien el primer día: quién es la empresa, cómo trabajamos, qué reglas seguimos. La diferencia es que la IA lo lee en milisegundos y lo recuerda siempre.

Sin CLAUDE.md

Empleado nuevo, primer día

  • ✕Poderoso pero genérico — no sabe nada de tu negocio
  • ✕Le explicás todo de cero en cada conversación
  • ✕Recomendaciones que no encajan con tu realidad
Con CLAUDE.md

Colaborador de meses

  • ✓Sabe quién sos y en qué trabajás desde el segundo cero
  • ✓Arranca con el contexto completo, sin repetir nada
  • ✓Habla tu idioma y tu tono — como Iván habla
2 · Cómo crear el Proyecto

4 pasos, una sola vez y queda armado

1
Creá la carpeta
Terminal de VS Code (Ctrl+ñ):
mkdir mi-proyecto → cd → git init
2
Registrala en VS Code
Con Project Manager tu proyecto queda a un click en el panel izquierdo. Sin buscar rutas.
3
Pegá el prompt base
En el panel Claude. Crea toda la estructura (memory, .claude, marca, marketing) sola. → próximo slide
4
Primer commit
git add .
git commit -m "init: estructura base"
terminal — pasos 1 y 4
$ mkdir mi-proyecto && cd mi-proyecto
$ git init # ← Paso 1
...pegás el prompt base en el panel Claude (Paso 3)...
$ git add . && git commit -m "init: estructura base" # ← Paso 4
3 · La plantilla del prompt base del negocio

Un prompt arma toda la estructura

Vamos a crear la estructura completa de un proyecto de IA profesional en la carpeta actual. Hacé TODO en orden: ## ESTRUCTURA DE CARPETAS memory/ ← progreso.md, errores-aprendidos.md, decisiones.md bitacora/ .claude/ ← commands/ y handoffs/ marca/ ← logo, icono, tipografia, colores, identidad.md marketing/ ← afiches, reels, carruseles, historias, copy/ ## CONTENIDO DE LOS ARCHIVOS BASE - memory/progreso.md → estado actual + ✅ completado + ⏳ pendiente - memory/errores-aprendidos.md → formato: fecha · descripción · causa · solución - memory/decisiones.md → por qué elegí cada herramienta o enfoque - marca/identidad.md → nombre, paleta, tipografía, tono, público ## REGLAS - Carpetas de recursos: creálas con un .gitkeep adentro - Todos los .md con encabezado: # título + > Estado: pendiente - Al terminar mostrá el árbol completo con ✅ en cada item - Decime qué tengo que hacer como próximo paso

5 · Para cerrar

Sin Proyecto vs con Proyecto

Sin Proyecto
  • ✕Archivos sueltos: notas, notas2, notas_final_ESTE...
  • ✕La IA trabaja en el caos — no sabe qué es importante
  • ✕Cero memoria entre sesiones — empezás de cero cada vez
  • ✕Reorganizar después cuesta el doble de tiempo
Con Proyecto
  • ✓Estructura clara: la IA sabe dónde está cada cosa
  • ✓El CLAUDE.md le da contexto desde el segundo cero
  • ✓Recuerda entre sesiones — el proyecto tiene historia
  • ✓5 minutos de estructura hoy = horas ahorradas después
Creá la estructura antes de empezar a trabajar.
